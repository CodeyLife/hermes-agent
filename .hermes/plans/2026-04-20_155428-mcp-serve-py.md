# Hermes MCP Server — 分析报告与优化计划

## 一、项目背景

Hermes Agent 是一个基于 LLM 的自治 Agent 系统，支持多消息平台（Telegram、Discord、Slack 等），具备完整的工具链、记忆系统、技能管理、会话管理等功能。

`mcp_serve.py` 作为 Hermes 的 **MCP 桥接服务层**，其核心目标：

- 让任意 MCP 客户端（Claude Code、Cursor、Codex、Trae 等）通过标准 MCP 协议与 Hermes Agent 交互
- 暴露 Hermes 的消息桥接能力（会话、消息、事件、审批）
- 暴露 Hermes 的本地学习资产（记忆、会话回忆、技能）
- 支撑 Trae 等客户端的 planner-executor 工作流（plan + task board）

## 三、架构设计亮点

1. **EventBridge 的 mtime 优化** — 200ms 轮询通过 `stat().st_mtime` 检查跳过未变化的文件，几乎零开销
2. **安全边界清晰** — `skill_view_safe` 明确不触发插件/外部查找副作用
3. **确定性搜索** — `session_recall_search` 不依赖 LLM 摘要，纯 SQLite FTS5 搜索
4. **上下文聚合** — `task_context_bundle` 将多个数据源整合，避免客户端多次调用
5. **Profile 隔离** — 通过 `get_hermes_home()` 自动支持多 profile 隔离
6. **队列限制与截断** — EventBridge 的 QUEUE_LIMIT=1000 防止内存泄漏

## 四、潜在问题与风险

### 4.1 中高风险

#### [P0] `messages_send` 无输入验证深度不足
**位置**: L796-826
**问题**:
- `target` 格式仅通过文档说明（"platform:chat_id"），无实际校验
- 未验证 `message` 长度限制
- 直接调用 `send_message_tool`，异常处理仅捕获通用 Exception

**改进**:
```python
# 添加格式校验
if ":" not in target:
    return json.dumps({"error": "target must be in 'platform:identifier' format"})

# 添加长度限制（参考各平台限制）
if len(message) > 4096:
    return json.dumps({"error": "message exceeds 4096 character limit"})
```

#### [P0] `memory_write` 缺少内容验证
**位置**: L897-923
**问题**:
- 未验证 `content` 长度或格式
- 无防注入检查（如恶意写入可执行内容）
- `replace` 动作的 `old_text` 未验证是否存在

**改进**: 添加内容长度限制（如 5000 字符）、格式验证、以及 replace 时 old_text 的存在性检查

#### [P1] EventBridge 审批状态内存-only
**位置**: L300-398
**问题**:
- `_pending_approvals` 是纯内存字典，重启后丢失
- `respond_to_approval` 标记为 "best-effort without gateway IPC"，实际可能无法真正通知 Gateway
- 无审批超时/过期机制

**改进**:
- 审批状态持久化到 SQLite
- 添加审批超时清理（如 5 分钟自动过期）
- 明确文档说明此限制

#### [P1] `skill_create_or_patch` 无权限/范围控制
**位置**: L975-998
**问题**:
- 客户端可以创建任意名称、任意类别的技能
- 无防覆盖保护（同名技能会被静默替换）
- 无操作审计日志

**改进**:
- 添加 `overwrite_protection` 参数（默认 True）
- 记录技能变更日志
- 限制可创建的类别白名单

### 4.2 低中风险

#### [P2] 会话数据加载效率
**位置**: L562, L609, L648, L703, L842
**问题**:
- 每次工具调用都重新 `_load_sessions_index()`（读文件+JSON 解析）
- EventBridge 有缓存但其他工具没有利用该缓存

**改进**:
- EventBridge 的 `_cached_sessions_index` 可以被其他工具共享
- 或使用 `functools.lru_cache` 装饰器

#### [P2] `messages_read` 无分页机制
**位置**: L636-686
**问题**:
- 只能取最近 N 条消息，无法向前翻页
- `limit` 最大无限制（可能返回过多数据）

**改进**: 添加 `offset` 参数或游标分页，添加 `limit` 上限（如 200）

#### [P2] `attachments_fetch` 仅提取 URL/路径，无内容获取
**位置**: L691-737
**问题**:
- 返回的是附件元信息（URL/路径），而非实际内容
- 如果 URL 已失效或路径不可访问，客户端无法直接获取附件

**改进**: 添加 `fetch_content: bool = False` 参数，支持可选地返回附件内容（base64 编码）

#### [P2] 错误响应格式不一致
**位置**: 多处
**问题**:
- 部分工具使用 `json.dumps({"error": "..."})`
- 部分工具使用 `_structured_error(...)`（包含 `success: false`）
- 客户端解析时需要处理两种格式

**改进**: 统一使用 `_structured_error()` 辅助函数

#### [P3] `_extract_message_content` 多部分内容提取不完整
**位置**: L133-142
**问题**:
- 只提取 `type == "text"` 的部分
- 不处理 `tool_result`、`reasoning` 等其他内容类型

**改进**: 根据实际需求扩展处理类型

#### [P3] `_get_session_db()` 每次调用创建新实例
**位置**: L86-93
**问题**:
- 每次工具调用都创建新的 `SessionDB()` 实例
- 虽然 SQLite 连接池开销小，但频繁创建不是最佳实践

**改进**: 使用单例模式或连接池

## 五、优化与改进计划

### 阶段 1：安全性与健壮性（P0）

1. **输入验证增强**
   - `messages_send`: 添加 target 格式校验、message 长度限制
   - `memory_write`: 添加 content 长度限制、格式验证
   - `skill_create_or_patch`: 添加类别白名单、覆盖保护

2. **错误响应格式统一**
   - 所有错误返回统一使用 `_structured_error()` 格式

### 阶段 2：性能优化（P1-P2）

3. **会话索引缓存共享**
   - 将 EventBridge 的缓存暴露给其他工具使用
   - 或使用 lru_cache 装饰器

4. **SessionDB 连接管理**
   - 实现单例/连接池模式
   - 减少重复连接创建开销

5. **messages_read 分页支持**
   - 添加 offset/游标参数
   - 添加 limit 上限保护

### 阶段 3：功能增强（P2-P3）

6. **审批状态持久化**
   - 将 `_pending_approvals` 写入 SQLite
   - 添加审批超时清理机制

7. **附件内容获取**
   - 添加可选的 `fetch_content` 参数
   - 支持 base64 编码返回附件内容

8. **操作审计日志**
   - 对所有写操作（memory_write, skill_create_or_patch, plan_update, task_board_update）添加日志记录

### 阶段 4：代码结构优化（P3）

9. **工具注册解耦**
   - 当前所有工具都在 `create_mcp_server()` 函数内定义
   - 建议按功能域分组到独立的注册函数中
   - 提高可维护性和可测试性

10. **测试覆盖增强**
    - 当前测试可能在 `tests/` 目录下
    - 需要确保每个 MCP 工具都有对应的测试用例

### 建议的工具分组结构

```python
def create_mcp_server(event_bridge: Optional[EventBridge] = None) -> "FastMCP":
    mcp = FastMCP("hermes", instructions=MCP_SERVER_INSTRUCTIONS)
    bridge = event_bridge or EventBridge()

    # 分组注册，提高可读性
    _register_messaging_tools(mcp, bridge)
    _register_memory_tools(mcp)
    _register_skill_tools(mcp)
    _register_context_tools(mcp)
    _register_plan_tools(mcp)
    _register_permission_tools(mcp, bridge)

    return mcp

def _register_messaging_tools(mcp, bridge):
    @mcp.tool()

    # ... 其他消息工具

def _register_memory_tools(mcp):
    @mcp.tool()
    def memory_read(): ...
    # ... 其他记忆工具
```

### 建议的输入验证装饰器

```python
def validate_input(**validators):
    """装饰器：在工具函数执行前验证输入参数"""
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for param, validator in validators.items():
                value = kwargs.get(param) or args[0]  # 简化逻辑
                if not validator(value):
                    return _structured_error(f"Invalid {param}")
            return func(*args, **kwargs)
        return wrapper
    return decorator
```
